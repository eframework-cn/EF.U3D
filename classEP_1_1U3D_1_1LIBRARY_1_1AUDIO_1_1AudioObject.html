Detailed Description

The object playing the audio clip associated with a AudioSubItem

If audio object pooling is enabled make sure you store references to an AudioObject by using PoolableReference<T>

var soundFX = new PoolableReference<AudioObject>( AudioController.Play( "someSFX" ) );
// some other part of the code executed later when the sound may have stopped playing
// and was moved back to the pool
AudioObject audioObject = soundFX.Get();
if( audioObject != null )
{
// it is safe to access audioObject here
audioObject.Stop();
}

Starts a fade-out. If the AudioItem mode is is a sequence, the next sub-item will continue to play after the this sub-item is completely faded out.

If the audio is already fading out the requested fade-out is combined with the existing one. This function only fades-out the primary audio source.

◆ FinishSequence()

void EP.U3D.LIBRARY.AUDIO.AudioObject.FinishSequence ( )

Finishes a playing sequence, depending on the AudioItem's loop mode

See also
AudioItem.LoopMode

:

  • AudioItem.LoopMode.LoopSequence: The sequence will stop after the current item has finished playing
  • AudioItem.LoopMode.PlaySequenceAndLoopLast: The sequence will stop after the current item has finished playing. If the sequence is during the looping part the looping will stop after the current loop reached its end.
  • AudioItem.LoopMode.IntroLoopOutroSequence: The sequence will stop after the current item has finished playing. If the sequence is during the looping part the outro will be played and the sequence will stop afterwards.

Has no effect if the audio is not in a sequence loop mode.

void EP.U3D.LIBRARY.AUDIO.AudioObject.SwitchAudioSources ( )

Switches the primary and secondary audio source.

This way a single AudioObject can play two audio clips at the same time. You can use it e.g. to cross fade between two audios using the same AudioObject.

playingAudioObject.FadeOut( 3 ); playingAudioObject.SwitchAudioSources(); playingAudioObject.PlayNow( "otherAudioID" ); playingAudioObject.FadeIn( 3 );

◆ volume

float EP.U3D.LIBRARY.AUDIO.AudioObject.volume
getset

Gets or sets the volume.

This is the adjusted volume value with which the Audio clip is currently playing. It is the value resulting from multiplying the volume of the subitem, item, the category, and the script parameter. It does not contain the global volume or the fading value.

"Adjusted" means that the value does not equal Unity's internal audio clip volume value, because Unity's volume range is not distributed is a perceptually even manner.

unityVolume = Mathf.Pow( adjustedVolume, 1.6 )
